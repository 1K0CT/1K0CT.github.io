CPython逆向

如果python程序中调用了一个打包成.pyd或者.so的外部连接库 那么那个库大概率就是Cython编译打包的 直接当成.dll或者.so来逆向

正式开始逆向之前需要先知道几个关于CPython的最基础的知识点 方便后续理解

marshal序列化对象逆向

我不采用任何加固 直接把源代码给你如何呢?

bytecode = marshal.dumps(obj)可以得到obj对象序列化后的字节码 常用的数据类型对象序列化后也能大概看出原型 但是如果是一个本身就长得很抽象的函数的字节码对象被序列化bytecode = marshal.dumps(func.__code__) 那就很烧脑了

好在这些字节码距离最常见的pyinstaller打包程序逆向中的.pyc(aka python bytecode)只差了一个魔术头 只要知道手上的字节码序列化自哪个版本的python并手动将该版本的魔术字填充到16字节((RAW_MAGIC_NUMBER.to_bytes(2, 'little') + b'\r\n').ljust(0x10, b'\x00'))修补到想反编译的字节码前就能用常用的反编译工具进行反编译了

但是 上面说的实际上通常是遇到类似逆向问题的最后一步 真正让我决定水一篇博客的实际上是各种对字节码的加密 真正的难点是拿到最终被marshal.loads(bytecode)反序列化的字节码 尤其是当解密过程涉及到自身函数元属性, 调用栈, 执行帧之类的运行时信息 解密就显得尤为困难了 这里介绍一种从一道题目学到的究极公式解法
